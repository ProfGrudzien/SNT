Le protocole IP

Un protocole pour se retrouver sur un réseau

Le protocole IP (Internet Protocole) permet de transmettre un paquet au bon ordinateur à travers le réseau. Le contenu du paquet n’a pas d’importance, seule compte l’adresse du destinataire et l’adresse de l’expéditeur. Sur un réseau informatique, les ordinateurs sont identifiés par une adresse appelée adresse IP. Ces adresses sont constituées de 4 nombres compris entre 0 et 255 pour la norme IPv4.

  1. Exécuter ce programme Python qui permet de déterminer l’adresse IP de votre ordinateur.

    # importer un module contenant des outils supplémentaires
    import os
    # utiliser un des outils du module os :
    os.system("ipconfig")

  2. Utiliser le site https://adresseip.com/ pour identifier l’adresse IP de votre ordinateur.
  3. A t’on obtenu la même adresse IP ?

Remarques :

  • L'adresse IP donnée par le programme Python donne l'adresse de l'ordinateur sur le réseau du lycée.
    Dans la classe, nous avons tous une adresse différente même si les deux premiers nombres sont identiques.
  • L'adresse IP donnée par le site internet donne l'adresse du routeur du lycée qui nous connecte à internet.
    Dans la classe, nous avons tous la même adresse.
Si j'essaye ces deux méthodes à la maison, je trouverai aussi deux adresse IP différentes : l'adresse de mon ordinateur sur le réseau local fourni par la box internet et l'adresse IP de ma box sur Internet.

Exemple d'utilisation de ces adresses :
Si j'installe un serveur Minecraft sur mon ordinateur, la première adresse me permet de rediriger correctement les ports dans les paramètres de ma box, la deuxième est celle que je communique à mes amis pour qu'ils me rejoignent.

Le routage

L’objectif est de permettre à deux machines du réseau de communiquer en passant par un chemin raisonnablement court, si possible optimal.

Le principe est de renseigner une table de routage à chaque nœud du réseau comportant l’adresse des autres nœuds directement connectés. Ensuite, de proche en proche, chaque nœud complète sa table en indiquant l'adresse du prochain intermédiaire à contacter pour communiquer avec les nœuds plus éloignés.

Bilan

Le protocole IP permet de gérer en temps réel les chemins empruntés par les paquets échangés sur le réseau en fonction des équipements disponibles. Il y a un « apprentissage » des routes prises par ces paquets. C’est donc un protocole très souple qui dépend très peu du réseau matériel. En revanche, ce protocole ne permet pas d’assurer que tous les paquets soient transmis ou qu’ils sont transmis dans le bon ordre. Le protocole TCP permet de combler ces lacunes.
