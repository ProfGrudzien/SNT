Adresse URL et serveur DNS

Les adresses URL

L’adresse IP est au centre du protocole TCP/IP et permet d’identifier le serveur ou l’ordinateur avec lequel on souhaite communiquer. Cette adresse est composée de 4 nombres entiers compris entre 0 et 255 pour la norme IPv4.

Cependant, de nos jours, nous n’avons pas besoin de connaître ces adresses IP pour accéder aux ressources désirées. A la place des adresses IP, nous utilisons une adresse URL (Uniform Resource Locator). Il s’agît d’une chaîne de caractères qui permet de localiser la ressource demandée. Elle est formée :

  • du nom du protocole utilisé pour traiter les données
  • d’un nom de domaine : un nom qui représente un ensemble de serveur
  • d’éventuels compléments pour identifier la ressource sur ce nom de domaine identifiables par le symbole « / »
  • d’éventuels informations pour détailler une requête identifiables par le symbole « ? »

Par exemple, l’adresse https://fr.wikipedia.org/wiki/Domain_Name_System est formé :

  • du nom du protocole https
  • du nom de domaine fr.wikipedia.org (qui est en fait un sous domaine de wikipedia.org)
  • des 2 compléments : wiki et Domain_Name_System
  • d’aucune information pour détailler une requête

Lorsque le nom du protocole n'est pas écrit, le protocole https est utilisé.

Identifier les différentes parties qui composent les adresses URL suivantes :
  1. https://www.netflix.com/browse
  2. https://www.youtube.com/results?search_query=dinosaure
  3. ftp://ftp.mozilla.org/pub/mozilla.org/firefox/releases/
  4. www.salto.fr/fargo-p_47441

Serveurs DNS

Il faut donc un lien entre les adresses IP des serveurs hébergeant les ressources et les adresses URL. Des serveurs (dont l’adresse IP est fixe, publique et largement répandu) font office d’annuaire. Ce sont les serveurs DNS.

Lorsque l’utilisateur saisi une adresse URL, ces serveurs sont interrogés afin d’obtenir une adresse IP associée. Dans le cadre d’une navigation habituelle, les adresses IP sont donc totalement invisible à l’utilisateur.

Sur Android, le serveurs DNS utilisé est celui de google, disponibles aux adresses IP 8.8.8.8 et 8.8.4.4. D’autres serveurs DNS fournissent aussi des services supplémentaires, comme par exemple openDNS qui permet des corrections automatiques des noms de domaine ou un blocage de domaine par catégories (pour la mise en place d’un contrôle parentale par exemple.)

Empoisonnement du cache DNS

L’empoisonnement du cache DNS ou pollution de cache DNS (DNS cache poisoning ou DNS cache pollution en anglais) est une technique permettant de leurrer les serveurs DNS afin de leur faire croire qu’ils reçoivent une réponse valide à une requête qu’ils effectuent, alors qu’elle est frauduleuse. Une fois que le serveur DNS a été empoisonné, l’information est mise dans un cache, rendant ainsi vulnérables tous les utilisateurs de ce serveur. Ce type d’attaque permet, par exemple, d’envoyer un utilisateur vers un faux site dont le contenu peut servir à de l’hameçonnage ou comme vecteur de virus et autres applications malveillantes.

Attaque par déni de service (DDoS)

L’attaque par déni de service consiste à submerger le serveur DNS par un nombre très important de demandes simultanées. L’objectif est alors de rendre inutilisable le service car le serveur n’est pas en mesure de répondre à toutes les requêtes. Pour parvenir à envoyer un grand nombre de demande, les pirates informatiques utilisent au préalable des virus pour infecter des milliers d’ordinateurs.
